# `news.ieumedu.kr` 운영 가이드

## 1. 현재 상태

이 폴더는 `이음통합평생교육원` 사이트와 분리된 인터넷신문 스테이징입니다.

- 교육원 본사이트: `ieumedu.or.kr/`
- 신문 스테이징: `news.ieumedu.kr/`

현재 GitHub Pages 워크플로는 `ieumedu.or.kr/`만 `_site`로 복사합니다.
즉, 이 폴더는 아직 자동 배포되지 않습니다.

## 2. 왜 분리했는가

인터넷신문 사이트는 아래 공개 구조가 필요합니다.

- 제호
- 발행인
- 편집인
- 주사무소/발행소
- 대표 전화 및 공식 이메일
- 등록번호
- 등록연월일
- 청소년보호책임자
- 기사형 목록/상세 구조
- 정정·반론·권리침해 접수 채널

현재 교육원 사이트는 수강안내·자격안내 중심이므로, 같은 화면 안에 신문사 구조를 섞으면 행정상·브랜딩상 모두 혼선이 큽니다.

## 3. 다음 단계

### 3-1. 등록 준비

최소한 아래를 먼저 확정해야 합니다.

- 제호 확정 및 중복 조회
- 발행인 실명
- 편집인 실명
- 주사무소 및 발행소
- 공식 대표 전화
- 공식 대표 이메일
- 청소년보호책임자
- 기사 운영 주제와 발행 목적 문안

### 3-2. 홈페이지 준비

초기 공개 페이지는 다음 순서로 확장합니다.

1. 메인
2. 기사 목록
3. 기사 상세
4. 회사소개/신문소개
5. 정정·반론·권리침해 안내
6. 청소년보호정책
7. 이용약관
8. 개인정보처리방침

현재는 `기사화-편성안.md`를 기준으로 블로그 아카이브 원고를 기사형으로 재구성합니다.
초기 공개면은 `생애주기와 수면`, `특수·통합교육`, `이론 브리프` 세 묶음을 우선 사용합니다.
원장 소개 자료 활용 여부는 `원장자료-기사화-검토.md`에서 별도로 분류합니다.
브런치북 원고의 칼럼 적합도는 `브런치북-칼럼-검토.md`에서 따로 관리합니다.

### 3-3. 배포 분리

`news.ieumedu.kr`를 실제로 쓰려면 배포도 분리하는 게 좋습니다.

권장안:

- 교육원: 현재 GitHub Pages 유지
- 신문: 별도 저장소 또는 별도 배포 프로젝트 사용

같은 저장소를 유지하고 싶다면:

- `news.ieumedu.kr`를 별도 브랜치 또는 별도 Pages 프로젝트로 분리
- 또는 Netlify/Vercel 등 별도 호스팅으로 연결

## 4. 등록 체크리스트

### 법정/행정 기본

- 제호 확정
- 발행목적 문안 확정
- 발행내용 문안 확정
- 주사무소 주소 확정
- 발행소 주소 확정
- 발행인 확정
- 편집인 확정
- 단체/법인 정관 또는 규약 정리
- 임차 시 임대차계약서 준비
- 발행인/편집인 기본증명서 준비

### 사이트 공개 구조

- 첫 화면에서 신문 제호 인식 가능
- 기사 목록 존재
- 기사 발행일 표시
- 신문 연락처 공개
- 발행인/편집인 공개
- 등록번호/등록연월일 반영 자리 확보
- 청소년보호책임자 공개
- 정정/반론/권리침해 접수 경로 표시

## 5. 주의사항

- 등록 전에는 등록번호나 등록연월일을 실제값처럼 표시하면 안 됩니다.
- 교육원 공지와 신문 기사를 같은 목록에서 운영하지 않는 것이 좋습니다.
- 교육원 홍보문과 기사문을 구분하지 못하면 인터넷신문형 공공성이 약해 보일 수 있습니다.
- 상담 사례 원문은 그대로 기사화하지 말고, 반드시 익명화·일반화한 뒤 기사 문법으로 재작성해야 합니다.
